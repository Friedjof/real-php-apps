Payload Dropzone

This HTTP server is part of a controlled hacking lab and is intended for serving files to targets during exploitation scenarios. Typical use cases include:

  • Dropping reverse shell payloads
  • Providing exploit scripts or binaries
  • Delivering post-exploitation tools
  • Transferring files between host and target containers

To serve a file, simply place it into the /web directory located beneath the application being tested. No configuration is needed — all files in that directory are made available via HTTP.

Access paths:

  • From your host browser: http://web.localhost/
  • From within other containers: http://172.103.0.3/<filename>

This setup allows fast and flexible delivery of arbitrary files during offensive testing without requiring changes to the target environment.
